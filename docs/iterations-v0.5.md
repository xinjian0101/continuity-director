# v0.5.0 迭代记录

1. 升级版本到 v0.5.0，Manifest Schema 升级到 1.4，并保留 1.0—1.3 自动迁移。

2. 新增声明式工作流模板校验，限制节点结构、类名、重复 ID 和占位符，禁止执行任意代码。

3. 新增工作流占位符绑定器，支持精确占位符保留原始类型、嵌入式字符串替换和缺失字段检测。

4. 新增运行配置快照，固定项目、序列、模型与工作流指纹，并自动清除 API Key、Token、密码等敏感字段。

5. 新增严格任务状态机，限制 queued、leased、running、succeeded、failed、cancelled、blocked 的合法迁移并记录历史。

6. 新增持久化队列标准化，将生成队列转换为带修订号、时间戳、任务指纹和运行快照关联的状态对象。

7. 新增依赖就绪任务领取与租约，按优先级确定性分配，避免多个工作进程重复执行同一任务。

8. 新增租约续期与过期回收，失联任务自动回队或在超过尝试上限后失败，支持崩溃恢复。

9. 新增原子队列存储，使用临时文件与 os.replace 防止半写入，并通过乐观修订号避免并发覆盖。

10. 新增生成素材索引，统一视频、图片、音频、字幕和元数据，并根据 SHA-256 识别重复产物。

11. 新增素材索引合并，防止跨项目混用，并合并重复素材的标签与元数据。

12. 新增质量门槛配置，覆盖身份、时序、动作、解剖、灯光、转场和技术质量，并支持权重与硬性阈值。

13. 新增 Take 质量评估，计算加权总分、硬性失败、警告和逐维度问题，形成机器可读验收结果。

14. 新增 Take 排名，优先通过质量门槛的结果，再按总分和问题惩罚确定性排序。

15. 新增逐镜头最佳 Take 选择，支持只接受完全通过门槛的结果，并明确列出仍需处理的镜头。

16. 新增自动重做计划，根据失败维度给出身份、时序、动作、灯光和转场调整建议，并限制每镜重做次数。

17. 新增可追踪运行日志，事件带顺序号、时间戳、级别和指纹，并可汇总错误、耗时与事件分布。

18. 新增可恢复运行包，将快照、队列、质量门槛、素材索引和追踪日志原子打包，并验证路径与 SHA-256 完整性。

19. 将工作流绑定、运行快照、持久队列、质量验收、Take 选择、自动重做、追踪和运行包暴露为 15 个 ComfyUI 节点。

20. 扩展回归测试、发布校验、示例与中英文文档，覆盖工作流、队列恢复、质量闭环、追踪和运行包完整性。
