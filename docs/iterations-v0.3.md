# v0.3.0 二十轮生产化迭代

本文件记录从 v0.2.0 到 v0.3.0 的连续二十轮有效迭代。

## 第2轮：结构化问题与多角色演员表
- 新增统一 issue 结构。
- 新增 cast_lock，可锁定多个角色并检测重复角色 ID。

## 第3轮：场景空间拓扑
- 新增场景分区、锚点和有向连接。
- 可检测人物在无路径情况下突然跨区。

## 第4轮：多角色对白校验
- 校验说话人是否存在、对白是否为空。
- 保存情绪与口型同步标记。

## 第5轮：分支剧情镜头链
- 可从任意父镜头生成独立分支。
- 每镜记录 branch_id、分支根指纹和分支序号。

## 第6轮：可控重试与 Take 变体
- 同一镜头可生成 1—32 个确定性 Take。
- 支持仅换种子或轻微构图变化，避免无规则重复抽卡。

## 第7轮：平台提示词与参考图预算
- 为不同适配器设置可配置的提示词、负面词和参考图上限。
- 超限时确定性截断并给出明确警告。

## 第8轮：自定义平台适配器配置
- 支持用户以 JSON 声明平台能力、预算和字段映射。
- 严格校验配置，不执行任意代码，也不保存密钥。

## 第9轮：平台能力协商
- 生成前明确平台支持的参考图类型、种子策略与传输方式。
- 不兼容项以结构化 issue 返回。

## 第10轮：内容寻址缓存
- 根据完整 JSON 内容生成 SHA-256 缓存键。
- 原子写入、重复结果复用、按数量安全清理。

## 第11轮：断点恢复
- 保存已完成与待生成镜头清单。
- 恢复时校验序列指纹，防止错接其他项目。

## 第12轮：项目包完整性验证
- 校验必需文件、SHA-256 和 ZIP 路径安全。
- 可识别损坏、缺失和被替换的镜头文件。

## 第13轮：序列自动修复
- 可预览修复建议，也可重建镜头序号、重复 ID 和父指纹链。
- 修复前后分别运行审计，避免静默改写。

## 第14轮：NDJSON 数据交换
- 支持整组镜头按行导出，便于版本管理和流式处理。
- 导入时逐行定位 JSON 错误并恢复 sequence_manifest。

## 第15轮：生产就绪报告
- 汇总连续性、时长、提示词、警告和平台能力差异。
- 输出 0—100 就绪评分与明确阻断项。

## 第16轮：生产功能接入 ComfyUI
- 新增演员表锁、Take 变体、序列修复、生产报告和项目包验证节点。
- 新功能可直接在节点图内组合使用。

## 第17轮：前端节点识别与版本同步
- 新节点加入专属颜色和搜索分组。
- 前端版本徽标升级为 v0.3.0，避免界面与后端版本不一致。

## 第18轮：生产功能测试矩阵
- 增加演员表、空间拓扑、分支、Take、预算、缓存、断点、校验、修复和 NDJSON 测试。
- 覆盖正常路径与压缩包篡改路径。

## 第19轮：Schema 1.2 向后兼容
- 补齐旧镜头的分支字段、Take 字段和旧序列默认值。
- 更新旧版迁移测试，使其验证当前 Schema，而不是写死历史版本。

## 第20轮：发布收口
- 更新中英文文档、Changelog 和发布校验规则。
- 发布校验要求至少 18 个节点，并强制检查全部生产节点映射。
- 完成 v0.3.0 二十轮迭代记录。
