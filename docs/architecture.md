# 架构说明

## 数据层次

```text
continuity_core.py
  项目、角色、场景、镜头、提示词、种子和跨镜状态
        ↓
production_core.py
  演员表、分支、Take、序列修复、项目包
        ↓
runtime_core.py
  参考帧、时间线、依赖图、模型能力和执行计划
        ↓
orchestration_core.py
  工作流绑定、持久化队列、租约、质量门槛、重做和运行包
        ↓
postprocess_core.py
  媒体探测、抽帧、技术质检、边界检查、拼接、版本治理和可观测性
        ↓
collaboration_core.py
  权限、编辑锁、审批、审计链、分布式节点、模板信任和重放验证
```

## 确定性原则

- 相同项目、镜头和 Take 输入生成相同种子。
- 对象使用规范化 JSON 计算内容指纹。
- 队列、重跑、配额预留和回滚要求修订号或指纹仍匹配。
- 已通过镜头不会因其他镜头失败而自动重做。
- 拼接前可以验证源文件 SHA-256。

## 外部程序

FFmpeg/FFprobe 是可选的本地系统工具，仅在用户显式执行视频探测、抽帧或拼接节点时调用。命令使用参数数组，不经过 Shell，不执行输入 JSON 中的命令。

## 外部质量模型

插件不内置人脸识别或身份模型。外部模型输出通过声明式字段路径、量纲和反向距离规则映射为 0—100 分；适配器不执行代码。

## 安全与恢复

- 运行包和配置包使用 ZIP 路径检查与 SHA-256。
- API Key、Token、Cookie、密码等字段会被清除。
- 版本快照可比较并生成受指纹保护的局部或完整回滚计划。
- 产物血缘图检测缺失父项、自引用和循环依赖。


## 协作一致性原则

- 写入操作使用修订号或租约令牌，防止旧状态覆盖新状态。
- 审批、变更请求和审计日志保存角色与动作来源。
- 审计事件通过前序哈希形成可验证链。
- 分布式调度只分配依赖已满足且能力兼容的任务。
- 模板包只做声明式校验，不执行包内代码。
